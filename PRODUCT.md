# Product

## Register

product

## Users

Interior designers and smart home integrators at Beyond Alliance use this to compose, review, and export keypad engraving layouts for luxury residential installations. The same session is then shared with the end client — a high-net-worth homeowner reviewing their smart home layout — who needs to understand, approve, or comment on it without any technical vocabulary. Both audiences are in the room at the same time.

## Product Purpose

Replace hand-drawn or spreadsheet-based keypad labelling with a fast, precise digital tool that turns a room list into a complete engraving file. Success means: integrator sets up a project in under 10 minutes, client approves it in one meeting, factory receives an unambiguous file.

## Brand Personality

Precise. Refined. Confident. The tool should feel like the smart home systems it specifies — premium hardware made simple. Not SaaS, not technical, not sterile. Closer to a well-designed Apple product page: breathing room, large confident type, obvious affordances, zero clutter.

## Anti-references

- Generic SaaS dark dashboards (Vercel clones, crypto tools)
- Busy CAD/AV software UIs (Crestron Toolbox, AMX)
- Flat corporate intranets
- Cream/sand AI-default warm palettes — the warmth should come from spacing and typography, not background tint

## Design Principles

1. **The tool disappears into the work.** Integrators should focus on keypad labels, not the software. Every chrome element earns its presence.
2. **Dual audience clarity.** What the integrator configures, the client must instantly understand. If a non-technical person can read the output confidently, it's right.
3. **Physical honesty.** The on-screen keypad should look and feel like the physical device being ordered. Colour, layout, engraving mode — everything is visible, nothing is abstract.
4. **Progressive disclosure.** Simple things are visible immediately. Power features (icons, engraving modes, per-button colours) reveal themselves as needed without cluttering the first view.
5. **Presentation-ready at any moment.** At any point in the workflow, switching to Client View should produce something you'd proudly show on a projector.

## Accessibility & Inclusion

WCAG AA. Full keyboard navigation for integrator workflows. Print stylesheet must produce clean black-on-white output regardless of screen theme.
